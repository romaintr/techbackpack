HISTOIRE INTERNET

Internet est né en 1969 sous le nom DARPANET

Le réseau devait assurer les échanges d'informations entre les centres névralgiques américains en cas d'attaque nucléaire soviétique.

Si l'un ou plusieurs des sites venait à être détruit, les messages parviendraient aux destinataires par des itinéraires alternatifs.

HISTOIRE INTERNET

Il devient en 1972 ARPAnet, il relie des centres universitaires. C’est l’ancêtre d’Internet.

Entre 1991 et 1993 à Genève, Tim Berners-Lee définit le protocole HTTP utilisé pour afficher les pages web Le Web est né et avec lui avec « Mosaic » le premier Navigateur Web.

A partir de 1995 Internet est devenu accessible au grand public.

Adresse IP

Une adresse IP est un numéro unique permettant de communiquer dans un réseau.

Il ne peut pas avoir plusieurs ordinateurs avec la même adresse IP dans un même réseau. On peut comparer l’adresse IP à un numéro de téléphone

Afin de facililter l'utilsisation du réseau, il existe un systéme d'adresse plus explicite qui sont les noms de domaines.

Protocole DNS

Le protocole DNS permet d'établir la correspondance entre le nom de domaine et l'adresse IP des machines d'un réseau via un serveur de nom de domaine

L'ICANN est l'organisme qui gére les noms de domaine

L'enregistrement d'un nom de domaine se fait via un REGISTAR

Consultation du registre : WHOIS
Exemple de registarwww.gandi.net

Protocole TCP/IP

TCP veut dire Transmission Control Protocole

TCP/IP représente l'ensemble des règles de communication sur internet afin de pouvoir acheminer des paquets de données d’un ordinateur à un autre, en s’appuyant sur les adresses IP des ordinateurs.

Protocole HTTP

HTTP a été inventé par Tim Berners-Lee avec les adresses Web et le langage HTML pour créer le World Wide Web.

Le protocole HTTP utilise le protocole TCP/IP pour permettre un transfert de fichiers (essentiellement au format HTML et des images ) entre un navigateur (le client) et un serveur Web

Le fichier à transférer est localisé grâce à une adresse appelée URL

Différence entre web et internet


Internet n'est rien d'autre qu'un réseau de transport de données. Le web est construit au dessus d'internet qui l'utilise pour transporter les données.

Différence entre web et internet

Plein d'autres applications que le web sont construites au dessus d'internet:
le mail (protocoles SMTP, POP3...)
le chat (protocoles Jabber, AIM...)
le transfert de fichier (protocoles FTP...)
les VPN (pour relier des réseaux informatique privés)

Le client: le navigateur web

Le but principal d'un navigateur est de récupérer depuis un serveur le fichier demandé par l'utilisateur et de l'afficher dans sa fenêtre.

Le client: le navigateur web

La façon dont le navigateur interprète et affiche les fichiers HTML est précisée dans les spécifications HTML et CSS.

Ces spécifications sont maintenues par le l'organisation W3C (World Wide Web Consortium), organisation des normes du Web.

Consultation du registre : WHOIS

Le client: le navigateur web

Les principaux composants d'un navigateur sont :
Le moteur de rendu = HTML+CSS
Le réseau: protocole HTTP
L'interpréteur JavaScript
Le stockage de données: les cookies

Le serveur

Le serveur HTTP est le logiciel prenant en charge les requêtes du protocole HTTP. Le plus connu est Apache

Le plus souvent, un serveur Web utilise d'autres logiciels qui fonctionnent en collaboration.

Certaines combinaisons de logiciels de base sont connues sous différents acronymes, notamment celle d'Apache (serveur HTTP), MySQL (serveur de base de données) et PHP (langage de programmation)

Développement front end VS back end

Ce sont deux profils types que l’on retrouve dans la plupart des agences web de nos jours.

Lorsque l’on parle de «Front-End», il s’agit des éléments du site que l’on voit à l’écran et avec lesquels on peut interagir.

Les éléments du développement front-end sont composés de HTML, CSS et de Javascript contrôlés par le navigateur web.

Développement front end VS back end

Le Back-End, c’est la partie du code qui est exécutée par le serveur permettant de générer une page HTML

Le travail du serveur sera codé dans un langage propre à lui : le langage serveur

Les données sont stockées dans une bases de données

Page statique

Ce n'est pas une page sans mouvements ou sans animations, mais une page visible telle qu'elle a été conçue. Ces pages peuvent présenter toute forme de contenu, images, musique, vidéo etc... mais elles sont toujours présentées de la même façon. Elles ne changent pas et c'est en ce sens qu'elles sont statiques.

Page dynamique

Elles permettent de présenter les informations de différentes manières selon l'interaction du visiteur, ou le contenu publié depuis un backoffice. Les pages sont alors construites "à la volée" grâce à une programmation conçue par le développeur. Le contenu est issu d'une base de données puis mis en page en temps réel.

C'est le cas par exemple d'un site d'E-commerce: présentation des articles par thèmes, couleurs, prix etc... C'est également le cas des blogs, des sites d'informations, des sites participatifs comme Facebook ou Instagram où les visiteurs génèrent le contenu du site.

Langages front-end

Le HTML ( Hypertext Markup Language) est un format de données conçu pour présenter les pages web. Le HTML permet de structurer sémantiquement le contenu d'une page web et d'inclure des ressources multimédias (images,vidéos,sons) ainsi que des formulaires

Le HTML ne permet pas de mettre en page le contenu, c'est le role du CSS qui est un langage permettant de gérer le style visuel ( le design de la page )
Le HTML et le CSS vont donc de pair.

Langages front-end

Le HTML5 et le CSS3 correspondent à la dernière version dont les spécifications sont à l'étude depuis 2008

Le HTML5/CSS3 permet d'intégrer des pistes audios et vidéos dans une page sans passer par Flash. Il permet également d'intégrer des animations 2D et 3D et de créer des jeux entièrement en HTML5. Il permet aussi un stockage des données pour un fonctionnement hors-connexion ce qui est essentiel pour construire une webapp

Langages front-end

Un dernier élément pour créer des pages interactives: le Javascript. Ces scripts sont gérés et exécutés par le navigateur web sans faire appel aux ressources du serveur.

  • Animer texte et image
  • Réagir aux événements de la souris
  • Réaliser des calculs
  • Gérer des menus


Essentiel pour développer une Web App et la rendre interactive

Langages front-end

Une Web App est une application web accessible via internet et n'a pas besoin d'être téléchargée. Généralement développée pour le mobile elle possède des fonctionnalités spécifiques au mobile grâce aux nouvelles fonctionnalités offertes par HTML5 (accés à la webcam, géolocalisation via GPS, accés aux photos ... )

Langages back-end PHP

PHP: Hypertext Preprocessor, plus connu sous son sigle PHP (acronyme récursif), est un langage de programmation libree principalement , utilisé pour produire des pages Web dynamiques via un serveur HTTP

PHP a permis de créer un grand nombre de sites web célèbres, comme Facebook, YouTube, Wikipedia, Google,[réf. nécessaire]. Il est aujourd'hui considéré comme la base de la création des sites Internet dits dynamiques

Le script php est executé sur le serveur et non sur le poste client (contrairement par exemple au javascript).

Langages back-end MYSQL

MySQL est un serveur de base de données. MySQL est un SGBD (Gestionnaire de base de données en français). Il permet de stoker des données en masse et pouvoir les utiliser à volonté en les triant comme on le souhaite. Le couple PHP/MySQL est très utilisé par les sites web et proposé par la majorité des hébergeurs Web.

Langages back-end : les templates

Un template (parfois appelé layout) est strucutre sans contenu dont les éléments statiques sont déjà positionnés ( header/footer/colonnes/textes ... )

Il peut y avoir plusieurs templates dans un site web : un pour la page d’accueil, un pour une page de contenu classique, un pour une fiche produit ....

Les templates sont au centre du fonctionnement des sites dynamiques.

Les CMS

L'acronyme CMS signifie Content Management System, pour système de gestion de contenu.
Les CMS offrent la possibilité de construire un site web à partir d’une interface de gestion simplifiée accessible depuis un backoffice

De cette manière, il est possible de maintenir et faire vivre un site web sans la moindre connaissance technique dans les métiers du Web.

Les CMS


L’édition des pages est réalisée avec des éditeurs WYSIWYG.
Les gestionnaires de contenu proposent généralement des modules de gestion des menus de navigation, des images et des fichiers.
La mise en page est réalisée à partir de modèles (templates) contenus dans des thèmes.
La chaine de publication (workflow) est gérée grâce à une hiérarchie d'utilisateurs, rôles et permissions (utilisateur anonyme, administrateur, contributeur, etc.) ;

Application mobile

  • 1993: IBM créé le premier smartphone
  • 1997: Démarrage du webmobile avec le WAP( Wireless Application Protocol)
  • 1999: Démarrage du Wifi et du HotSpot
  • 2007: Apple lance l'IPHONE
  • 2008: Apple lance l'Appstore
  • 2008: Sortie des premiers téléphones sous Android
  • 2010: Apple lance l'IPAD

Application native

  • Logiciel sur mobile ou tablette utilisable avec ou sans connexion internet
  • Téléchargeable depuis des stores: Apple Store, Google Play
  • Ces applications sont conçues pour plateforme grâce au kit de développement SDK
  • Chaque kit de développement utilise un langage différent
  • Apple :SDK IOS 5 , langage Objective-C
  • Google: SDK Android , langage java

Web app

  • Application utilisée via Internet sur le navigateur web de l'appareil
  • Pas besoin d'être téléchargée ou installée pour fonctionner
  • Développée avec des langages webs: HTML5/CSS/Javascript

Application hybride

L'application hybride dite 'encapsulée' correspond à la fusion de l'application mobile native et la web app

La capsule est développée dans le langage natif de l'appareil ( Java, Objective C ). Une webapp HTML5 est injectée dans l'application du téléphone

L'application est ainsi disponible sur tous les différents stores

Application hybride

  • Une code unique pour plusieurs plateformes: IOS/ANDROID/WINDOWS
  • Baisse du cout du développement et de maintenance
  • Pour accéler le développement des frameworks sont disponibles: Phonegap
  • La webapp et la capsule peuvent communique ensemble: GPS,PHOTOS, CONTACT....
